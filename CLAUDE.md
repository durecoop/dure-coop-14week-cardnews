# 두레생협 주간 할인 생활재 카드뉴스 프로젝트

## 개요
매주 두레생협 온라인몰 추천 페이지의 할인 상품을 모바일 카드뉴스(weekly_deal.html)로 제작하는 프로젝트.

## 주간 카드뉴스 생성 프로세스

사용자가 "N주차 카드뉴스 만들어줘"라고 요청하면 아래 단계를 순서대로 수행한다.

### STEP 1: 상품 데이터 수집
- **소스**: https://ecoop.or.kr/DureShop/recommend.do
- WebFetch로 페이지 크롤링 (여러 번 나눠서 전체 상품 빠짐없이 수집)
- **주의**: WebFetch는 한 번에 전체 데이터를 못 가져올 수 있음. 반드시 "1~25번", "26~50번", "50번 이후" 등으로 나눠서 수집하고 총 개수를 확인할 것
- 수집 항목: 상품명, 용량/규격, 조합원 원가, 할인가, 할인율(%), 구매중 수량, **이미지 파일명(확장자 .jpg/.png 정확히)**, 상품 설명
- **노출 순서**: 쇼핑몰 페이지 위에서부터 그대로 유지 (임의 정렬 금지)
- **이미지 URL 패턴**: `https://dureimg.ecoop.or.kr:9091/Delsys/DLOG/Goods/GoodsMaster/GoodsImage/{ID}.jpg|png`
  - 확장자가 .jpg인지 .png인지 반드시 페이지에서 확인! 잘못된 확장자 = 이미지 깨짐
- 같은 상품의 사이즈/색상 변형(예: 허브염색약 3종, 반팔티 S/M/L)은 하나로 통합

### STEP 2: 상품별 콘텐츠 작성
각 상품에 대해:
1. **f1 (특징 1줄)**: 원재료/산지/인증 중심 팩트 — 페이지에서 추출한 설명 텍스트 활용
2. **f2 (특징 2줄)**: 활용법/생활 밀착 제안
3. **rv (리뷰 1줄)**: 조합원 후기 톤으로 작성
4. **tags**: 무항생제, 무농약, 유기농, 국산, 제철, 동물복지, 기획세트 등 해당하는 것만

### STEP 3: HTML 생성 (weekly_deal.html)
- weekly_deal.html 파일을 **전체 덮어쓰기** (CSS/JS 구조는 동일하게 유지)
- 변경할 부분:
  1. **커버 슬라이드**: 주차 번호, 할인 품목 수, 최대 할인율, 지금 주문 중 합계
  2. **페이지 인디케이터**: `1 / (상품수+1)`
  3. **products 배열**: 전체 상품 데이터 교체
  4. **rvData**: 인기 상품 7~10개에 리뷰 요약 데이터 추가 (buying 수가 많거나 할인율 높은 상품)

### STEP 4: 배포
```bash
git add weekly_deal.html
git commit -m "N주차 할인 생활재 데이터 업데이트 (XX개 상품)"
git push origin master
```
- GitHub Pages 자동 반영 (1~2분 소요)
- 라이브 URL: https://durecoop.github.io/dure-coop-14week-cardnews/weekly_deal.html

### 카피 톤
- 생협 감성: "생산자의 정성", "얼굴 있는 먹거리"
- 짧고 감성적: "봄바람과 함께 찾아온 부산의 선물"
- 구매 유도: "이번 주, 닭볶음탕 어떠세요?"
- 페이지에서 추출한 상품 설명 텍스트를 f1에 적극 활용

### 인터랙션
- 세로 스와이프 (릴스/숏폼 스타일)
- 터치 + 마우스 드래그 + 키보드 ↑↓
- 상단 페이지 인디케이터 (1 / N)
- 가이드 오버레이 (첫 진입 시 "밀어서 생활재 보기")

## 검수 체크리스트
- [ ] 상품 개수가 실제 페이지와 일치하는지 (WebFetch 여러 번 호출해서 확인)
- [ ] 이미지 확장자(.jpg/.png)가 페이지 원본과 일치하는지
- [ ] 할인율·가격이 조합원가 기준인지 (비조합원가 X)
- [ ] 구매중 수량이 최신인지
- [ ] 커버 슬라이드의 주차·품목수·최대할인율이 맞는지

## 구매수량 자동 업데이트
- `update_buying.py` 스크립트가 매시 50분에 ecoop 페이지에서 구매수량 크롤링
- 이미지 파일명(예: 56862C)을 고유 키로 매칭하여 HTML의 buying 값 업데이트
- Windows 작업 스케줄러: `EcoopBuyingUpdate` 태스크
